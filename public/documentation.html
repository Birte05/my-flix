Documentation of the My-Flix-App

This web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Request Endpoint URL and query format Method type Format of Data expected in the body of the request Format of the response data
Show root page / GET none Text
Return a list of all movies to the user /movies/ GET none A JSON object holding data about all the movies. Example:

{ Title: {type: Sring, required: true}, Description: {type: String, requried: true}, Genre: { Name: String, Description: String }, Director: { Name: String, Bio: String }, Actors: [String], ImmangePath: String, Featured: Boolean }
Return data about a single movie by title /movies/[Title] GET none A JSON object holding data about a single movie, containing a title, description, genre property, director property and image. Example:

{ Title: {type: Sring, required: true}, Description: {type: String, requried: true}, Genre: { Name: String, Description: String }, Director: { Name: String, Bio: String }, Actors: [String], ImmangePath: String, Featured: Boolean }
Return data about a genre by name /movies/genre/[Title] GET none A JSON object holding data about a genre by name and description. Example:

{ Title: {type: Sring, required: true}, Genre: { Name: String, Description: String } }
Return Data about a Director by name /movies/Director/[Name] GET none A JSON object holding name and bio of the director. Example:

{ Director: { Name: String, Bio: String }, }
Return a list of all users /users/ GET none A JSON object holding data about all the users. Example:

{ ID: Integer, Username: String, Passowrd: String, Email: String, Birthday: Date }
Register a new user /users/ POST A JSON object holding data about the user to add, structured like:

{ ID: Integer, Username: String, Passowrd: String, Email: String, Birthday: Date }

A JSON object holding data about the user that was added, including an ID:

{ ID: Integer, Username: String, Passowrd: String, Email: String, Birthday: Date }
Allow user to update their user info /users/[Username] PUT A JSON object holding data about the user to add, structured like:

{ Username: String,(required) Password: String, (required) Email: String, (required) Birthday: Date

A confirmation stating that the user's information was updated.
Allow a user to add a movie to their list of favorites /users/[Username]/Movies/[MovieID] POST none A confirmation stating that the movie was added.
Allow users to remove a movie from their list of favorites /users/[username]/Movies/[MovieID] DELETE none A confirmation stating that the movie was deleted form the list.
Deregister a user /users/[username] DELETE none A confirmation stating that the user's account was deleted.
